安裝與設定APC

Jul 10th, 2008

安裝與設定APC
———–
linux:
可至PECL下載APC

下載APC後

# tar xvzf APC-2.0.3.tgz 解壓縮
# cd APC-2.0.3
# /usr/bin/phpize
# ./configure –enable-apc
# make
# make install

windows:
可在PECL4Win下載編譯好的dll
請選擇適當版本自行丟到php include目錄中
———-

接著更改php.ini設定

貼上
linux:
extension = apc.so

win:
extension = apc.dll

apc.shm_segments=1
apc.optimization=0
apc.shm_size=128
apc.ttl=7200
apc.user_ttl=7200
apc.num_files_hint=1024
apc.mmap_file_mask=/tmp/apc.XXXXXX
apc.enable_cli=1
apc.rfc1867 = on

重開apache2ctl graceful後，就安裝好了。
可至phpinfo.php檢查是否有正確啟動
或用壓縮檔中的apc.php查看apc資訊

上傳大檔案

Jul 9th, 2008

因為公司有要作上傳大檔案的需求
這兩天研究了一下，得到了幾個結論

1.HTML file input和flash都沒辦法做到續傳，因為沒辦法指定從檔案的哪個位置開始讀取。若要做到續傳，可能要配合java或activeX。

2.網路上的許多上傳套件，只要有用flash接的，一概不能用。因為flash FileReference有100mb的限制。

3.目前以純PHP來說，要做到真實的上傳比例進度棒，是不可行的。因為無法在上傳完畢以前就取得檔案大小以及目前檔案大小(用來算比例)。因此大多配合flash來取得檔案大小。但不適用於本專案(有大檔案需求)。

4.有一個套件APC(Alternative PHP Cache)，可以解決真實上傳比例進度這個問題。但是這個套件現在沒有包含在標準安裝裡面(PHP 6應會內建)，現階段需要自行下載安裝。
20080710更新：利用APC實做upload progress無法multithread，等於無用！需另尋方法。

5.真實網路情況下，用http post傳大檔案(以500mb作測試)，還是相當不穩定且速度緩慢。

Look Cleats

Jun 17th, 2008

之前被鞋底板的問題困擾，因此在網路上問了一下
同時也做了一點survey…

Look的鞋底板(Cleats) 現在有分兩種。
分為Keo和Keo以前的，兩者不通用。

Keo的Cleats現在也有兩種款式，
一種是傳統平滑型，
另一種則有加上防滑片。
兩種通用。
有分紅色9度、灰色4.5度以及黑色0度三種。

而Keo以前的Cleats則叫做Look Delta。
分為紅色9度以及黑色0度兩種。


有Free ARC系統的Look卡踏(如我使用的pp396)
需要搭配紅色9度的Cleats
才能使用Free ARC的旋鈕調整0度、3度、6度以及9度四段可旋轉角度(angular float)。
若有Free ARC的踏板搭配黑色0度Cleats，
則Free ARC會失效，不管怎麼調整都是0度。
 
 
現在的Look keo系列似乎已經取消Look ARC的設計
改用鞋底板來調整。
我推測應該是為了減輕踏板重量的關係…。


拿之前買的Exustar Cleats出來比一比…



可以明顯看出有許多的不同，
下方曲線設計差異很大，
更別說長度整整短了一節。
難怪騎一騎會滑脫出來。

這個故事告訴我們：
鞋底板還是買原廠的好啊…。XD

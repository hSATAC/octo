有 Bug!

現在的公司，有一件事讓我很不能適應。

他們很喜歡說「有 Bug」。

在這邊，他們會用 Bug 來統稱一切不合意的狀況，
諸如：這頁想要和另一頁調換、需求修改等等…統統叫做 Bug。
真正的 Bug 佔他們回報的 Bug 數量大概十分之一吧。

基本上，我聽到「有 Bug」這句話會緊張，
當然不是因為我寫的程式不會有 Bug，難免都會有的。
但是當一直說有 Bug ，結果卻發現是需求大改，
而這些規則早已和企劃確認過無數次。

這樣的狀況讓我很緊繃。

真希望能夠修改歸修改、需求歸需求、Bug歸Bug…

PHP 自動 Urldecode $_GET

最近碰到一個問題，當 url 內含+號 (plus sign)時
PHP會自動轉換成空白，做了一點調查才知道，

似乎 PHP 5 之後 PHP會自動 urldecode QUERY_STRING
照理講根據RFC 1738 +號不是unsafe的，可以留著
但是PHP的urlencode和rawurlencode有些許不同，

rawurlencode是根據RFC 1738所以加號不會處理掉
而 PHP 自己的 urlencode 會把空白 (space) 轉成 + 號 (plus sign)
然後到 PHP 這邊自己會 urldecode 出來 + 號就變成空白了。至於
為什麼會這樣，PHP網站也語焉不詳，只說了一句 “historical reason”
事實上似乎 ASP 和 JAVA 也是這樣處理 urlencode 的
目前解法大概是
1.可能會有 + 號的地方用 urlencode 包一下(有用 base64_encode 的地方)
2.或是在接可能有 + 號的地方再 urlencode 一次
3.有的framework不使用$_GET 而自己去 parse query_string

Login SFTP With Www-data

之前 ubuntu 上沒灌 ftpd, 最近習慣 scp 以後，
也想幫自己的主機設定一下，
不然之前改一些小檔案都要 ssh 進來 vim 完後還要 chown 十分瑣碎。

研究了一下，才發現原來 ssh 內建 sftp 了。
那要怎樣使用 www-data 登入呢？
一開始本想設定群組和有效群組，但後來覺得實在太麻煩。

觀察了一下 www-data 的資料，他的 home 目錄是設在 /var/www
那就直接 mkdir 一個 .ssh 然後把 authorized_keys 丟進去，
用 scp 一連，果然順利登入了，非常方便。
